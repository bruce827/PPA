# 反盗油专项系统方案

本方案针对单个采油厂规模，设计一套以"销量与库存异常偏差预警"为核心的反盗油专项工作系统。通过在重点管道、监测井场、站场外输管道等关键节点部署硬件传感器，建立产-销-存全链条数据监控和自动报警体系，实现对盗油行为的智能预警和精准打击。

本方案是[[反盗油系统设计]]的专项落地版本，技术原理参见[[管道泄漏监测技术]]和[[智能监控与预警技术]]。关键数据来源包括[[原油自动交接技术]]（精确外输计量）和[[大罐自动盘库技术]]（实时库存监测），三者联动构成完整的物料平衡监控链。

## 三、销量与库存异常偏差预警系统

### 3.1 核心逻辑

偏差预警的核心思路是建立"产量 — 库存 — 销量（外输量）"的物料平衡模型：

```
单井产量之和 + 期初库存 = 外输量 + 期末库存 + 正常损耗

当  |实际偏差| > 阈值  时，触发报警
```

**正常损耗**包括：管道蒸发损耗、计量误差、温度体积修正等，根据历史数据统计确定基准值。

### 3.2 数据来源方案

#### 方案 A：对接已有系统

适用于已建设 SCADA/ERP/计量系统的油田，通过接口获取数据：

| 数据项 | 来源系统 | 对接方式 | 数据频率 |
|--------|---------|---------|---------|
| 单井产量 | SCADA / 计量系统 | OPC UA / API | 实时 / 小时级 |
| 储罐液位 | SCADA / [[大罐自动盘库技术]] | OPC UA / Modbus TCP | 实时（10 秒级） |
| 管道流量 | SCADA / RTU | OPC UA | 实时（秒级） |
| 外输计量 | [[原油自动交接技术]] / 计量系统 | API / 数据库 | 实时（秒级） |
| 销售交割量 | ERP / 财务系统 | API / 数据库 | 日级 |
| 装卸记录 | 拉运管理系统 | API | 事件触发 |
| 储罐油层厚度 | [[大罐自动盘库技术]] | OPC UA / Modbus TCP | 实时（10 秒级） |
| 储罐分层含水 | [[大罐自动盘库技术]] | OPC UA / Modbus TCP | 实时（10 秒级） |

**对接要点**：
- 统一时间戳（NTP 时钟同步）
- 统一计量单位（标准条件下的体积/质量）
- 数据质量校验（缺失值、跳变、传感器故障判断）

#### 方案 B：传感器原始采集

适用于信息化程度较低的油田，从传感器直接采集数据：

```
传感器 → RTU/采集终端 → 边缘网关 → 数据中心 → 分析引擎
         (协议转换)     (数据压缩)   (存储清洗)  (偏差计算)
```

**采集参数要求**：

| 数据项 | 采集频率 | 存储 | 精度 |
|--------|---------|------|------|
| 井口流量 | 1 次/分钟 | 小时均值 + 分钟级原始 | ±2-5% |
| 储罐液位 | 1 次/秒 | 秒级原始 + 分钟均值 | ±1mm |
| 管道压力 | 10 次/秒 | 秒级原始（负压波分析） | ±0.075%FS |
| 管道流量 | 1 次/秒 | 秒级原始 | ±0.2% |
| 站场进出站量 | 1 次/秒 | 秒级原始 | ±0.15% |

### 3.3 偏差分析模型

#### 模型一：产量-外输偏差

```
偏差率 = (Σ单井产量 - 外输量 - 正常损耗) / Σ单井产量 × 100%
```

- 时间窗口：班级（8小时）、日级
- 正常偏差范围：考虑计量误差叠加，基准约 ±2-3%
- 动态基线：根据近 30 天数据滚动计算历史均值和标准差

#### 模型二：库存异常偏差

```
库存偏差 = 实际液位变化量 - 预期液位变化量

预期液位变化 = 产入量 - 外输量 - 装卸量 - 正常损耗
```

大港油田通过[[大罐自动盘库技术]]已实现 37 具沉降罐 10 秒级实时盘库，液位精度 ±1%，油层厚度精度 ±1%，为库存偏差模型提供了高精度实时数据基础。

- 实时监控每个储罐的液位变化曲线
- 自动比对液位变化与进出油记录
- 识别非正常的液位下降（排除正常作业后）

#### 模型三：管道输入输出偏差

```
管段偏差率 = (进站总量 - 出站总量) / 进站总量 × 100%
```

- 实时计算每条管道的输入输出平衡
- 结合负压波监测结果交叉验证
- 区分泄漏和盗油：泄漏通常持续微量，盗油通常突发大量

#### 模型四：时序异常检测

- 对每个监控点的关键指标建立时序基线
- 使用统计方法（3σ准则）或机器学习方法检测异常点
- 综合多个指标的异常信号，降低误报

### 3.4 报警规则设计

#### 分级标准

| 等级 | 颜色 | 偏差率阈值 | 持续时间 | 推送对象 | 处置时限 |
|------|------|-----------|---------|---------|---------|
| 一级预警 | 黄色 | 3%-5% | 持续 2 小时以上 | 中心站值班人员 | 4 小时内 |
| 二级报警 | 橙色 | 5%-10% | 持续 1 小时以上 | 调度室 + 中心站 | 2 小时内 |
| 三级报警 | 红色 | >10% 或突发性大偏差 | 即时 | 作业区经理 + 调度室 + 公安联动 | 即时响应 |

#### 复合条件报警

单一指标偏差可能由计量误差等正常原因导致，以下复合条件触发报警更可靠：

| 复合条件 | 含义 | 报警等级 |
|---------|------|---------|
| 产量偏差 + 液位异常 | 产量正常但库存异常下降 | 直接三级 |
| 管道偏差 + 负压波报警 | 流量不平衡且有泄漏信号 | 直接三级 |
| 产量偏差 + 夜间 + 可疑车辆 | 多因素叠加 | 直接三级 |
| 连续多日一级偏差 | 持续性小额偏差，疑似缓慢盗油 | 升至二级 |
| 液位突降 + 无装卸记录 | 非正常排液 | 直接三级 |

### 3.5 自动报警流程

```
传感器/系统数据实时采集
        ↓
   偏差计算引擎（分钟级轮询）
        ↓
   ┌─ 偏差率 < 3% → 正常，记录数据
   ├─ 3% ≤ 偏差率 < 5% → 计时，持续 2h 触发一级
   ├─ 5% ≤ 偏差率 < 10% → 触发二级
   └─ 偏差率 ≥ 10% → 触发三级
        ↓
   复合条件叠加判定（是否升级）
        ↓
   推送通知（短信/APP/大屏）
        ↓
   值班人员确认 → 现场核实 → 处置记录
        ↓
   超时未处置 → 自动升级推送上级
        ↓
   闭环：报警 → 核实 → 处置 → 归档 → 分析
```

## 五、部署计划

### 阶段一：基础设施搭建（1-3 月）

- 站场进出站口高精度流量计安装
- 站场储罐导波雷达液位计安装
- 数据汇聚平台部署
- 与 SCADA/ERP 系统接口对接（方案 A 场景）
- 或传感器直接采集网络搭建（方案 B 场景）

### 阶段二：偏差预警引擎上线（3-5 月）

- 物料平衡模型部署
- 偏差计算引擎上线
- 报警规则配置与调试
- 大屏看板和 APP 推送上线

### 阶段三：井场监控覆盖（5-8 月）

- 井口流量计和压力传感器安装
- 井场视频监控系统部署
- 井场无线通信网络建设
- 单井产量纳入偏差计算

### 阶段四：管道泄漏监测（8-10 月）

- 重点管道压力变送器部署
- 负压波泄漏监测系统上线

### 阶段五：优化完善（10-12 月）

- 电子执勤系统升级
- 复合条件报警规则优化
- AI 异常检测模型训练
