# PPA 2.0 文档目录

本目录用于存放 `PPA 2.0` 的独立产品规划与需求文档。

## 使用原则

- 与现有 `docs/prd/` 完全分离
- `docs/prd/` 继续承载 `1.0` 已有能力、历史设计与未完成事项
- 本目录只讨论 `2.0` 的版本目标、产品定位、功能规划与实现方案

## 建议放置内容

- `product-brief.md`
- `prd.md`
- `architecture.md`
- `epics-and-stories.md`
- 其他与 `2.0` 直接相关的专题文档

## 当前定位

`PPA 2.0` 建议作为“平台化第一步”的版本空间，重点验证：

- 公开项目池
- 多人参与评估
- 基础激励机制
- 初步商业化闭环

## PRD 结构

`prd.md` 是当前 V2 综合 PRD，按三大方向组织：

1. 数据与存储底座迁移：当前优先方向，覆盖 SQLite 到 Supabase PostgreSQL、项目附件到 Supabase Storage、Git/文件安全治理、验证与回滚。
2. SaaS 平台规划及补充：承接公开项目池、多人评估、聚合结果、奖励、付费报告和运营后台。
3. 其它内容整合：承接商业模式、投资人叙事、定价与奖励实验、问答材料等规划资产。

详细索引和阅读顺序见 `prd.md` 的 `0.4` 节。
