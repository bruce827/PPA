## 字段决策规则库整理

基于真实数据统计，规则分三层：**硬规则**（可程序化校验）、**软规则**（AI辅助判断）、**冲突规则**（发现异常需人工确认）。

### 第一层：控件选型规则（硬规则）

**规则1：数字类字段 → 必须用数字框**

字段类型为 `整型 / 长整型 / 单精度数字 / 双精度数字` → 输入类型必须是 `数字框`。数据中出现过数字框用于字符类型（长度1、20），这属于填写错误，需标记。

**规则2：日期/时间类字段 → 控件枚举判断**

|字段类型|长度|推荐控件|备注|
|---|---|---|---|
|日期|10|日期选择器|仅日期无时分秒|
|日期时间|19|日期时间选择器|含时分秒|
|日期 / 日期时间|任意|日期区间|过滤器场景专用|
|日期|10|年份选择器|仅年份精度|
|字符|20|年月选择器|年月精度|

**规则3：字符/文本类字段 → 长度决定控件**

|字段类型|长度范围|推荐控件|说明|
|---|---|---|---|
|字符|≤ 255|文本框|标准单行|
|字符|> 255 且无输入参数|文本域|多行文本|
|文本|任意|文本域|text类型默认文本域|
|字符/文本|任意|下拉选择器|有输入参数dict引用时|
|字符/文本|任意|下拉树形选择器|有输入参数dict引用且是树形数据时|

> 数据中存在字符1000用文本框的情况，这是潜在填写错误，需软规则提示。

**规则4：有输入参数 → 控件必须是选择类**

输入参数非空 → 输入类型必须在 `下拉选择器 / 下拉树形选择器 / 弹出选择器 / 复选框 / 单选框` 中，否则参数无效。

**规则5：上传类控件 → 字段类型约束**

`图片上传 / 文件上传 / 附件上传` → 字段类型必须是 `字符(255)` 或 `文本(1000)`，存储路径或URL。

---

### 第二层：显示控制规则（软规则，AI辅助）

**规则6：主键字段 → 四场景全隐藏**

是否主键=是 → 新增/更新/详情/列表控制全部为隐藏，默认值通常是系统生成。

**规则7：系统字段 → 全隐藏**

字段编码属于 `creator / updater / dept_id / tenant_id / deleted / create_time / update_time` → 卡片分组建议为"系统信息"，四场景全隐藏。

**规则8：只读字段的列表排序**

列表控制=只读 且 列表排序为空 → 提示补充列表排序，否则列表列顺序不确定。

**规则9：过滤器字段的一致性**

是否过滤器=是 → 过滤方式必须非空，过滤器提示信息建议填写。

---

### 第三层：异常检测规则（数据中已发现的真实问题）

**异常1：字符1000用文本框**（数据中出现8次） 字段类型=字符 且 长度≥500 且 输入类型=文本框 → 疑似应为文本域，建议确认。

**异常2：过滤方式出现中英文混用**（数据中有 `eq` 和 `等于` 并存） 过滤方式应统一使用编码（eq/like等），中文值需修正。

**异常3：日期类字段用时间选择器**（部分日期字段用了时间选择器） 字段类型=日期(长度10) + 输入类型=时间选择器 → 语义不一致，需确认是否需要时间精度。

**异常4：列表控制出现Excel公式引用**（`=表单字典值!$J$2:$J$5`） 少数行的列表控制列未正确填入中文值，而保留了Excel公式，导入时会报错。
