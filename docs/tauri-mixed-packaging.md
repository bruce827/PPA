## 方案概述

### 什么是 Tauri 混合方案？

**Tauri 混合方案 = Tauri（前端壳）+ Node.js（后端）**

```
前端：Tauri WebView（系统浏览器）
后端：独立 Node.js 进程（你的 Express 代码）
通信：HTTP（http://localhost:3001）
```

### 为什么是"混合"？

| 组件 | 技术栈 | 原因 |
|------|--------|------|
| **前端 UI** | Tauri WebView | 使用系统浏览器，不打包 Chromium |
| **后端逻辑** | Node.js + Express | 完全复用现有代码，零改动 |
| **数据库** | Supabase（云端）+ SQLite（可选） | 云端数据 + 本地文件 |
| **文件存储** | 本地磁盘（userData） | 解决文件服务问题 |

### 核心优势

| 优势 | 说明 | vs Electron |
|------|------|-------------|
| ✅ **体积更小** | ~30MB（节省 70MB） | **小 70%** |
| ✅ **启动更快** | ~2s（节省 1s） | **快 33%** |
| ✅ **内存更少** | ~150MB（节省 150MB） | **少 50%** |
| ✅ **代码复用** | 前后端几乎不改 | 相同 |
| ✅ **易于传播** | 直接发安装包 | 相同 |

## 核心原理

### 1. Tauri 的本质

**Tauri 不是一个完整的浏览器，而是一个"窗口管理器"**

```
Electron：完整的浏览器 + Node.js
├── Chromium（50MB）
├── Node.js（30MB）
└── 你的代码（20MB）

Tauri：窗口管理器 + 系统浏览器 + Node.js（可选）
├── Tauri 运行时（5MB，Rust 编写）
├── 系统 WebView（0MB，Windows/macOS/Linux 自带）
├── Node.js（10MB，可选，仅混合方案）
└── 你的代码（15MB）
```

### 2. 系统 WebView 是什么？

Tauri 使用操作系统自带的浏览器引擎：

| 平台 | WebView 实现 | 浏览器引擎 | 版本 |
|------|-------------|-----------|------|
| **Windows 10+** | WebView2 | Chromium（Edge） |  evergreen（自动更新） |
| **macOS** | WebKit | WebKit（Safari） | 系统版本 |
| **Linux** | WebKitGTK | WebKit | 系统版本 |

**关键点**：
- ✅ **不需要打包浏览器**：用户电脑上已有
- ✅ **自动更新**：系统浏览器会随系统更新
- ⚠️ **依赖系统版本**：Windows 7 不支持（但 Win 10/11 都有）

### 3. 两个独立的 V8 引擎

Tauri 混合方案中，**有两个独立的 V8 实例**：

```
┌─────────────────────────────────────────┐
│  前端（WebView）                         │
│  ├── V8 引擎 #1（或 JavaScriptCore）     │
│  └── 运行 React + UmiJS 前端代码         │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  后端（Node.js）                         │
│  ├── V8 引擎 #2                          │
│  └── 运行 Express 后端代码               │
└─────────────────────────────────────────┘
```

**为什么有两个 V8？**
- WebView 的 V8 用于渲染 UI（前端）
- Node.js 的 V8 用于运行服务端逻辑（后端）
- **两者完全独立，通过 HTTP 通信**

  ├── Windows：创建 WebView2 控件
  ├── macOS：创建 WKWebView
