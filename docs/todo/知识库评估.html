第二部分：五维度评分量表（1-5分制）

2.1 业务复杂度评估

评分 业务场景数量 规则变化频率 决策链长度 瑞云案例
1分-简单 单一业务场景 静态规则，年更 1-2步决策 不适用
2分-较简单 2-3个场景 规则变化少，季更 3-4步决策 不适用
3分-中等 3-5个场景 定期调整，月更 5-6步决策 部分适用
4分-较复杂 5-8个场景 动态变化，周更 7-9步决策 设备管理模块
5分-极复杂 8+场景 高度动态，随时调整 10步+，多层级 ✓ 投资审核模块

2.2 数据质量评估

评分 结构化程度 标注状态 质量评分 瑞云案例
3分-中等 半结构化为主 需清洗和标注 70-80分 ✓ 历史项目文档

2.3 技术难度评估

评分 技术方案 准确率要求 响应时间 瑞云案例
4分-较难 领域微调+RAG+知识图谱 95-98% 秒级 ✓ 需可解释AI

2.4 集成复杂度评估

评分 集成系统数量 接口协议 等保要求 瑞云案例
4分-较复杂 5-8个系统 多协议混合 等保三级 ✓ ERP+设备管理系统

2.5 定制化程度评估

评分 定制类型 规则引擎 可视化 瑞云案例
4分-深度 重度业务定制 低代码规则引擎 复杂可视化看板 ✓ 投资规则配置化
⚠️ 瑞云项目综合评分：
业务复杂度(5) + 数据质量(3) + 技术难度(4) + 集成复杂度(4) + 定制化程度(4) = 平均分4.0/5.0（高复杂度）
对应预算区间：400-550万（大型项目）

# 项目评估决策流程图 - Python代码 import matplotlib.pyplot as plt import matplotlib.patches as mpatches from matplotlib.patches import FancyBboxPatch, FancyArrowPatch fig, ax = plt.subplots(figsize=(18, 14)) ax.set_xlim(0, 18) ax.set_ylim(0, 14) ax.axis('off') colors = { 'start': '#27ae60', 'process': '#3498db', 'decision': '#f39c12', 'end': '#e74c3c' } # 绘制流程节点（简化示意） def draw_box(x, y, width, height, text, color): box = FancyBboxPatch((x-width/2, y-height/2), width, height, boxstyle="round,pad=0.1", facecolor=color, edgecolor='black', linewidth=2) ax.add_patch(box) ax.text(x, y, text, ha='center', va='center', fontsize=10, fontweight='bold', color='white') # 决策节点（菱形） def draw_diamond(x, y, size, text, color): diamond = plt.Polygon([(x, y+size), (x+size*1.2, y), (x, y-size), (x-size*1.2, y)], facecolor=color, edgecolor='black', linewidth=2) ax.add_patch(diamond) ax.text(x, y, text, ha='center', va='center', fontsize=9, color='white', fontweight='bold') # 绘制主要流程 draw_box(2, 11.5, 2.5, 0.8, '接收项目\\n需求文档', colors['start']) draw_box(5.5, 11.5, 2.5, 0.8, '第一阶段\\n业务理解', colors['process']) draw_diamond(9.5, 11.5, 0.6, '业务场景\\n清晰?', colors['decision']) draw_box(13, 11.5, 2.5, 0.8, '第二阶段\\n数据盘点', colors['process']) # ... 更多节点和箭头连接 ax.text(9, 13.5, 'AI知识库项目评估决策流程', ha='center', fontsize=20, fontweight='bold') plt.savefig('decision_flow.png', dpi=150, bbox_inches='tight')

人力成本计算公式： 人力成本 = Σ(角色人天 × 人天单价 × 投入比例) 人天单价参考： • 普通开发工程师：1,500-2,000元/人天 • AI算法工程师：2,500-3,500元/人天 • 知识图谱工程师：2,000-2,800元/人天 • 产品经理/项目经理：1,800-2,500元/人天 投入比例： • 全职投入：100% • 重度参与：80% • 中度参与：50% • 轻度参与：30% 示例（瑞云项目）： AI算法专家(2人) = 2 × 35,000元/月 × 90% × 10月 = 63万

第六部分：实施检查清单（Checklist）

6.1 评估阶段检查清单

第一阶段：项目启动准备

  • 项目需求说明书（含业务场景）
  • 现有系统架构图和技术栈清单
  • 数据资产清单（数据库、文档、接口）
  • 用户角色和权限需求
  • 安全合规要求（等保/分保）

第二阶段：数据资产评估

  • 历史数据总量估算
  • 数据增长速率（日/月增量）
  • 数据完整性检查（缺失率）
  • 数据准确性抽样验证
  • 标注需求和工作量估算

第三阶段：技术方案评估

  • 大模型选择（通用/垂直/自研）
  • 知识表示方案（向量/图谱/混合）
  • POC验证（准确率基线测试）
  • 性能指标确认（响应/并发）
  • 技术风险识别和应对

第四阶段：实施计划制定

  • MVP范围确定（最小可行产品）
  • 功能优先级排序（MoSCoW）
  • 分期交付计划（3-4期）
  • 每期里程碑和验收标准
  • 客户配合资源确认

6.2 风险识别检查清单

风险类别 风险描述 影响程度 应对措施
技术风险 大模型准确率不达预期 高 POC验证+规则引擎兜底+人工复核
需求风险 业务规则复杂且变化频繁 高 配置化机制+派驻运营人员
数据风险 历史数据质量不足 高 数据增强+外部标准库+人工标注
集成风险 与现有系统集成复杂 中 前期接口调研+预留适配层
资源风险 高端AI人才招聘困难 中 提前锁定+高校合作+外包

6.3 合同条款检查清单

  • 范围边界明确：详细列出包含和不包含的功能模块
  • 验收标准量化：准确率>95%、响应时间<3秒等可测量指标
  • 付款里程碑：建议3:3:3:1（启动:一期:二期:终验）
  • 变更管理流程：需求变更计价标准（人天单价×工作量）
  • 知识产权约定：代码归客户，技术框架可复用
  • 保密协议：数据安全条款、保密期限、违约责任
  • 质保金预留：建议预留10%，验收后1年支付
  • 等保合规责任：明确测评费用由谁承担
