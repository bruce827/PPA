# 待推送招标-全网检索功能交付报告

## 文档信息

- 模块：项目机会 / 待推送招标 / 全网检索
- 交付日期：2026-03-19
- 状态：已完成

## 本次交付范围

本次交付完成了“待推送招标 -> 全网检索”完整闭环，包括：

1. 联网搜索能力配置
2. 待推送招标页面内的单次全网检索
3. 检索结果保存、复用与安全刷新

## 交付功能点

### 1. 模型配置支持联网搜索能力

- 模型配置新增 `supports_web_search` 能力标记
- 支持在模型应用管理页面中维护、展示、查询该能力
- 后端支持按 `supports_web_search`、`is_active` 过滤模型
- 运行时会校验模型是否支持联网搜索

### 2. 提示词管理支持联网搜索模板

- 提示词模板新增 `web_search` 分类
- 前后端分类枚举、表单、展示、数据库约束全部打通
- 保持变量定义能力可继续复用
- 运行时会校验模板是否属于 `web_search`

### 3. 待推送招标新增“全网检索”入口

- 在待推送招标操作列新增“全网检索”按钮
- 点击后在当前页面打开弹窗，不跳转独立页面
- 弹窗会先加载当前招标上下文
- 若已有保存结果，优先展示最近一次保存结果

### 4. 弹窗内支持检索配置与页面级复用

- 支持选择联网搜索模型
- 支持选择 `web_search` 分类模板
- 支持返回条数、补充关键词、排除关键词
- 模型与模板默认值会在当前页面生命周期内复用

### 5. 已接通真实联网搜索执行链路

- 新增 `POST /api/opportunity/tender-staging/:id/web-search`
- 已接入运行时模型/模板校验
- 已接入提示词渲染与变量注入
- 外部联网搜索超时统一为 `180` 秒
- 对返回内容执行结构化 JSON 校验
- 对结果进行基础高相关过滤与排序

### 6. 检索结果展示可直接用于人工判断

- 结果采用“摘要优先 + Collapse 下钻”展示
- 首层可查看：
  - 网站名称
  - 页面标题
  - 发布时间
  - 来源类型
  - 置信度
- 展开后可查看：
  - 相关内容摘要
  - 关联原因
  - 网站链接
- 网站名称和外链均为新标签页打开，不影响当前弹窗和列表现场

### 7. 结果保存与再次打开复用

- 新增独立结果表 `tender_staging_web_search_results`
- 仅保存最近一次成功的结构化结果
- 不保存外部原始全文响应
- 再次打开同一条招标时，可直接读取最近一次保存结果

### 8. 历史结果与新结果状态表达

- 弹窗已明确区分：
  - 已保存结果
  - 本次新结果
- 已前置展示最近检索时间
- 时间格式统一为 `YYYY-MM-DD HH:mm`

### 9. 重新检索安全覆盖语义

- 当已有结果时，主按钮会显示为“重新检索”
- 重新检索成功：
  - 覆盖旧结果
  - 仍只保留 `1` 份当前有效结果
- 重新检索失败：
  - 不覆盖旧结果
  - 用户仍可继续查看上一份成功结果
