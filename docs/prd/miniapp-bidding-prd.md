# 招标快报小程序 PRD

**版本**: v1.0
**状态**: ✅ 已实现（独立部署）
**最后更新**: 2026-04-26

---

## 1. 产品概述

招标快报是一个用于业务验证的微信小程序。用户通过微信分享卡片进入小程序，查看当天或指定日期范围内的招标信息，并对感兴趣的项目执行"采纳"。

该产品不负责采集招标信息，数据由后续独立推送程序写入小程序数据库。首版目标是验证两个核心假设：

1. 通过微信分发，用户是否愿意查看推送的招标信息
2. 用户是否会对感兴趣的招标项目执行"采纳"

## 2. 产品目标

### 2.1 业务目标

1. 让招标信息从 PPA 数据源顺畅分发到微信用户
2. 建立最小可用的"查看 + 采纳"闭环
3. 记录具体由哪个微信用户采纳了某条招标信息

### 2.2 产品目标

1. 用户可通过微信登录进入小程序
2. 用户默认看到当天招标列表
3. 用户可按日期范围筛选招标信息
4. 用户可查看详情与招标公告
5. 用户可对未被采纳的项目执行采纳
6. 用户可取消自己发起的采纳

## 3. 用户定义

### 3.1 目标用户

- 微信用户
- 通过转发分享或主动进入小程序查看招标信息的人

### 3.2 用户权限

- 所有微信登录用户都可查看
- 所有微信登录用户都可对未被采纳的项目执行采纳
- 仅采纳人本人可取消自己的采纳

## 4. 适用场景

1. 某用户收到小程序分享卡片，打开后查看当天招标列表
2. 用户按日期范围筛选查看近期招标
3. 用户点进详情页，阅读项目基本信息与招标公告
4. 用户对感兴趣的项目点击"采纳"
5. 其他用户再次查看该项目时，看到"已被某人采纳"

## 5. MVP 范围

### 5.1 In Scope

1. 微信登录
2. 招标列表页
3. 日期范围筛选
4. 招标详情页
5. 采纳操作
6. 取消采纳操作
7. 采纳人展示
8. 按 `source_item_id` 去重，仅保留最新一条
9. 招标公告原始 HTML 存储
10. 招标公告简化富文本展示
11. 分享后打开"当天招标列表页"

### 5.2 Out of Scope

1. 招标信息采集
2. 复杂权限体系
3. 评论、收藏、多维筛选、关键词搜索
4. 推送订阅通知
5. 任务分配、跟进备注、多人协作审批流
6. 独立 CRM 或招投标管理系统能力

## 6. 页面结构

### 6.1 登录页

- 完成微信身份获取，建立小程序用户记录
- 首次进入时完成微信登录，缺少展示名称可提示补录

### 6.2 招标列表页

- 默认展示当天招标数据
- 支持日期范围筛选
- 列表字段：项目名称、发布日期、截止日期、招标单位、预算金额
- 状态展示：未采纳 / 已被某用户采纳

### 6.3 招标详情页

- 字段：项目名称、发布日期、截止日期、招标单位、预算金额、地区、来源平台、原文链接、项目摘要、招标公告、采纳状态、采纳人、采纳时间
- 操作：采纳 / 取消采纳

## 7. 业务规则

### 7.1 采纳规则

1. 一条招标只能被一个用户采纳
2. 若当前未被采纳，任意登录用户都可点击采纳
3. 采纳成功后记录采纳用户身份与采纳时间
4. 若当前已被采纳，其他用户只能查看，不能重复采纳
5. 仅采纳人本人可取消采纳
6. 取消采纳后，状态恢复为未采纳

### 7.2 去重规则

- 招标数据的业务唯一标识为 `source_item_id`
- 若推送程序两次推送同一 `source_item_id`，系统只保留最新版本

## 8. 技术实现

### 8.1 架构

- **后端**：CloudBase 云函数（非主 PPA server）
- **数据库**：CloudBase MongoDB 集合（`tenders`、`miniapp_users`、`miniapp_memberships`）
- **前端**：微信小程序（`frontend/ppa_miniapp/`）

### 8.2 核心 CloudFunctions

| 云函数 | 功能 |
|--------|------|
| `loginUser` | 微信用户登录 |
| `getTenderList` | 查询招标列表，支持日期范围筛选 |
| `getTenderDetail` | 获取招标详情，含会员访问控制 |
| `adoptTender` | 采纳招标（原子事务，防并发） |
| `cancelTenderAdoption` | 取消采纳（仅采纳人可操作） |
| `upsertTenderBySourceId` | 写入推送数据 |
| `seedSampleTenders` | 种子测试数据 |

### 8.3 与主 PPA 的关系

- miniapp 的 `tenders` 集合与主 PPA 的 `opportunity_tender_staging` 表**独立运行**
- 数据推送需通过 `upsertTenderBySourceId` 手动同步

## 9. 成功标准

1. 微信用户可顺利登录并进入招标列表
2. 默认当天列表可正常展示
3. 日期范围筛选可正常工作
4. 详情页可正常打开
5. 采纳与取消采纳状态准确
6. 页面能明确显示当前由谁采纳
7. 重复推送的同一数据不会重复显示
