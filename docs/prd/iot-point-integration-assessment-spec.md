# IoT 点位对接评估 V1.8 规格

**版本号:** V1.8  
**状态:** 已实现  
**适用范围:** 标准项目评估 `/assessment/new` 的工作量估算步骤  
**最后更新:** 2026-05-07

---

## 1. 背景与目标

IoT 点位对接项目的工作量主要来自点表整理、协议适配、点位映射、数据清洗、告警配置、批量导入、现场联调和验收核对。没有明确点表时，评估只能做准入判断，无法形成可靠报价；V1.8 聚焦“已有明确规模参数或点表口径”的手动估算。

V1.8 目标：
- 在标准 SaaS 评估流程中补充 IoT 点位对接评估能力，不新增独立项目类型。
- 以已有 IoT 平台/网关能力为前提评估点位接入实施成本，不评估 IoT 平台建设成本。
- 使用固定 8 项工作包模型实时计算建议人天，允许评估人逐项调整。
- IoT 点位对接成本独立进入总览报价，不并入系统对接成本。
- 保存完整估算依据，内部/外部 Excel 导出可追溯。

## 2. 非目标

- V1.8 不支持上传点表自动解析。
- V1.8 不按逐个点位逐行估算。
- V1.8 不提供 IoT 平台建设、网关硬件采购、网络施工、弱电施工、现场布线报价。
- V1.8 不新增独立风险系数；风险溢价沿用标准项目评估的 Rating Factor。

## 3. 用户流程

1. 用户进入 `/assessment/new`。
2. 在“工作量估算”步骤切换到 `IoT点位对接` Tab，Tab 文案前不显示图标。
3. 系统默认展示 8 项工作包，全部规模参数未填写时建议人天与调整后人天按 0 统计。
4. 用户填写规模参数，可填写点位总数，也可填写设备台数作为兜底估算依据；系统实时刷新建议人天。
5. 用户检查估算依据、建议人天，必要时修改调整后人天、角色和调整说明。
6. 用户点击“确认用于 IoT 独立报价”，系统记录确认时间；未确认前不参与总览报价。
7. 用户进入“生成总览”，点击计算最新报价。
8. 总览展示 IoT 点位对接成本和 IoT 点位工作量。
9. 保存项目后，`assessment_details_json` 保存 IoT 规模参数、工作包、估算点位数和确认时间。

## 4. 规模参数

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| 点位总数 | 否 | 优先作为估算点位数；未填写时可由设备台数折算 |
| 设备台数 | 否 | 未填写点位总数时，按每台设备 10 个点位兜底估算 |
| 站点/区域数 | 否 | 影响现场联调工作量 |
| 网关数量 | 否 | 记录规模依据，V1.8 不直接进入公式 |
| 协议类型数 | 否 | 影响协议/驱动配置工作量 |
| 存在私有协议 | 否 | 存在时增加协议/驱动配置工作量 |
| 控制点数量 | 否 | 记录点位复杂度，V1.8 不直接进入公式 |
| 告警点数量 | 否 | 告警规则默认可参考告警点数量 |
| 告警规则数量 | 否 | 影响告警规则配置工作量 |
| 高频采集点数量 | 否 | 记录采集压力，V1.8 不直接进入公式 |
| 清洗/换算点数量 | 否 | 影响数据清洗与单位换算工作量 |
| 计算/衍生点数量 | 否 | 影响数据清洗与单位换算工作量 |
| 历史存储点数量 | 否 | 记录存储范围，V1.8 不直接进入公式 |
| 需要现场联调 | 否 | 关闭时现场联调工作量为 0 |
| 现场联调次数 | 否 | 与站点/区域数取较大值 |
| 验收抽检比例 | 否 | 影响验收核对工作量 |
| 建筑面积/场地规模说明 | 否 | 仅作为估算说明，不直接进入公式 |

## 5. V1.8 8 项工作量模型

| 工作包 | 默认角色倾向 | 建议人天规则 |
| --- | --- | --- |
| 点表整理 | 实施工程师 | `估算点位数 > 0 ? 0.5 + ceil(估算点位数 / 500) * 0.5 : 0` |
| 协议/驱动配置 | 后端开发 | `协议类型数 * 1 + 私有协议附加 3` |
| 点位映射 | 后端开发 | `ceil(估算点位数 / 500) * 1` |
| 数据清洗与单位换算 | 后端开发 | `ceil((清洗/换算点数量 + 计算/衍生点数量) / 200) * 1` |
| 告警规则配置 | 后端开发 | `ceil(告警规则数量 / 100) * 1` |
| 批量导入 | 实施工程师 | `估算点位数 > 0 ? 0.5 + ceil(估算点位数 / 1000) * 0.5 : 0` |
| 现场联调 | 实施工程师 | `需要现场联调 ? max(现场联调次数, 站点/区域数) * 1 : 0` |
| 验收核对 | 实施工程师 | `估算点位数 > 0 且抽检点数 > 0 ? max(0.5, ceil(估算点位数 * 验收抽检比例 / 100) / 100 * 0.5) : 0` |

规则要求：
- 建议人天是系统基准值。
- 调整后人天是最终计价输入。
- 规模参数变化后，确认状态必须失效；未手动调整的工作包跟随建议人天实时更新，已手动调整的工作包保留调整后人天。
- 角色优先匹配名称包含 `实施` 或 `后端` 的角色；保存角色不存在时，后端兜底到当前可用角色。
- IoT 工作包不单独设置交付系数；交付难度通过“调整后人天”和标准风险评分因子体现。

## 6. 成本计算口径

IoT 工作包成本：

```text
工作包成本 = 调整后人天 × 角色日单价 × Rating Factor
```

IoT 点位对接成本：

```text
IoT点位对接成本 = sum(8项工作包成本)
```

总成本：

```text
总成本 =
软件研发成本
+ 系统对接成本
+ IoT点位对接成本
+ 差旅成本
+ 运维成本
+ 风险成本
```

系统对接成本不得包含已确认的独立 IoT 点位对接成本。历史数据兼容规则：如果旧项目只有 `integration_workload` 中的 `module1 = IoT点位对接` 行，且没有已确认的独立 `iot_point_integration.applied_at`，系统继续按旧系统对接口径计价，避免历史项目成本归零。

## 7. 数据结构

标准评估数据新增可选字段：

```json
{
  "iot_point_integration": {
    "assumptions": {
      "has_iot_platform": true,
      "includes_platform_build": false,
      "risk_factor_source": "standard_assessment"
    },
    "scale_params": {},
    "generated_items": [],
    "estimated_point_count": 0,
    "estimated_by_device_count": false,
    "applied_at": "ISO-8601"
  }
}
```

`generated_items` 单项字段：

```json
{
  "key": "point_mapping",
  "package_name": "点位映射",
  "estimate_basis": "按 1800 点映射到平台统一点位模型，每 500 点 1 天。",
  "suggested_days": 4,
  "adjusted_days": 4,
  "role_name": "后端开发",
  "adjustment_note": ""
}
```

`POST /api/calculate` 响应新增字段：

```json
{
  "iot_point_integration_cost": 0,
  "iot_point_integration_workload_days": 0
}
```

## 8. 总览与导出

总览必须展示：
- 软件研发成本 / 研发工作量
- 系统对接成本 / 对接工作量
- IoT 点位对接成本 / IoT 点位工作量
- 差旅成本
- 运维成本 / 运维工作量
- 风险成本
- 实施成本
- 总工作量
- Rating Factor 与风险得分上限

内部版 Excel 必须包含：
- Summary 中的 IoT 点位对接成本。
- `IoT点位对接` Sheet：评估前提、规模参数、估算点位数、风险评分因子、IoT 成本、IoT 工作量。
- `IoT工作包明细` Sheet：工作包、估算依据、建议人天、调整后人天、角色、调整说明、汇总行。

对外版 Excel 必须：
- 将已确认的 IoT 点位对接作为独立模块参与报价分摊。
- 如果 `role_costs` 已包含 IoT 模块，不得重复追加 `iot_point_integration.generated_items`。

## 9. 验收标准

1. 在 `/assessment/new` 工作量估算步骤可以看到纯文字 Tab：`IoT点位对接`。
2. 页面默认展示 8 项工作包；所有规模参数为 0 时，建议人天和调整后人天均按 0 展示。
3. 用户填写点位总数或设备台数后，8 项工作包建议人天实时刷新。
4. 修改规模参数、调整后人天、角色或调整说明后，确认状态失效；未重新确认前不参与总览计算。
5. 总览计算后，已确认的 IoT 成本与 IoT 工作量独立展示，系统对接成本不包含 IoT 成本。
6. 保存项目后，重新打开项目或模板时，IoT 评估数据可回显。
7. 内部版 Excel 包含已确认 IoT 评估的概要和工作包明细。
8. 对外版 Excel 不重复计入 IoT 模块。
9. 旧项目只有系统对接中的 IoT 行时，历史成本不丢失。
