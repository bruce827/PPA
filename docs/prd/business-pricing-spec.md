# 商务报价功能详细规格

**版本**: v1.0  
**最后更新**: 2026-03-30

---

## 1. 功能概述

商务报价功能是在既有“项目实施成本评估”之上的后置商务加工层。系统保持原有成本评估算法不变，仅在项目完成成本评估并保存后，允许用户基于该项目的实施成本生成一份**单轮商务报价快照**，用于内部商务测算、对外商务报价导出和历史回看。

本功能覆盖三个主要入口：

1. **参数配置页**：维护两种商务报价模式的默认费率/成本结构。
2. **项目历史页 / 详情页**：对已完成评估的项目触发商务报价、查看状态与结果。
3. **商务版 Excel 导出**：导出商务报价概览与模块明细。

### 1.1 设计原则

- **不改原成本算法**：`POST /api/calculate` 继续只负责项目实施成本评估。
- **报价后置生成**：只有在项目已有实施成本评估结果后，才允许生成商务报价。
- **单快照模式**：当前只保存每个项目的最新一版商务报价，不做多轮报价历史管理。
- **模式显式切换**：用户可在 `B端定制项目` 与 `企业级产品` 两种报价模式之间切换。

### 1.2 非目标

- 不支持多轮报价版本管理。
- 不在“新建评估”向导中直接嵌入商务报价步骤。
- 不改变历史项目原始实施成本数据。

## 2. 功能入口

### 2.1 参数配置页

- **路径**: `/config`
- **Tab 名称**: `商务报价配置`
- **用途**: 维护商务报价默认参数，不参与原始实施成本评估。

### 2.2 项目历史页

- **路径**: `/assessment/history`
- **入口**:
  - 操作列 `商务报价` 按钮
  - 导出下拉中的 `导出Excel（商务）`
  - 列表筛选项 `商务报价状态`
  - 列表字段 `商务报价总计（万元）`

### 2.3 项目详情页

- **路径**: `/assessment/detail/:id`
- **入口**:
  - 头部 `商务报价` 按钮
  - 头部 `导出Excel（商务版）`
  - 头部商务报价摘要区

## 3. 用户场景

### 3.1 场景 A：配置默认商务报价参数

1. 用户进入 `/config`。
2. 切换到 `商务报价配置` Tab。
3. 修改默认费率或默认成本结构。
4. 保存后，新的默认值会作用于之后打开的商务报价弹窗。

### 3.2 场景 B：在历史项目页生成商务报价

1. 用户进入 `/assessment/history`。
2. 从列表中选择一个已完成成本评估的项目，点击 `商务报价`。
3. 系统弹出商务报价弹窗，自动读取该项目的实施成本基线和默认配置。
4. 用户选择报价模式并调整参数。
5. 点击保存后，系统生成并覆盖保存该项目的最新商务报价快照。

### 3.3 场景 C：导出商务版 Excel

1. 用户在历史页或详情页点击导出。
2. 选择 `导出Excel（商务）`。
3. 系统基于已保存的商务报价快照输出商务版 Excel。
4. 如果当前项目尚未生成商务报价，则商务版导出入口不可用或阻止导出。

## 4. 参数配置页详细规格

### 4.1 页面结构

`商务报价配置` Tab 由两组配置组成：

1. **B端定制开发默认参数**
2. **企业级产品默认成本结构**

页面采用双列布局，桌面端每行两个控件，移动端自动折叠为单列。所有控件使用滑块组件，宽度 100%。

### 4.2 B端定制开发默认参数

| 字段 | 含义 | 区间 | 默认值 |
| --- | --- | --- | --- |
| `tax_rate` | 税率 | 6% - 13% | 6% |
| `management_rate` | 管理分摊率 | 10% - 15% | 12% |
| `sales_rate` | 销售商务率 | 10% - 15% | 12% |
| `profit_rate` | 利润率 | 10% - 20% | 15% |

### 4.3 企业级产品默认成本结构

| 字段 | 含义 | 区间 | 默认值 |
| --- | --- | --- | --- |
| `rd_rate` | 研发成本（R&D） | 30% - 40% | 35% |
| `cac_rate` | 营销与获客成本（CAC） | 30% - 50% | 40% |
| `cogs_rate` | 基础设施成本（COGS） | 10% - 20% | 15% |
| `csm_rate` | 客户成功与运维（CSM） | 10% - 15% | 10% |

### 4.4 配置校验规则

#### 4.4.1 B端定制项目

- 每个字段必须是合法数字。
- 每个字段必须落在各自定义区间内。

#### 4.4.2 企业级产品

- 每个字段必须是合法数字。
- 每个字段必须落在各自定义区间内。
- `R&D + CAC + COGS + CSM` 必须严格等于 `100%`。
- 若合计不为 `100%`，前端与后端都必须阻止保存，并提示：`企业级产品成本结构合计必须为 100%`。

### 4.5 配置接口

```http
GET /api/config/business-pricing
PUT /api/config/business-pricing
```

**响应结构**

```json
{
  "data": {
    "custom_development": {
      "tax_rate": 6,
      "management_rate": 12,
      "sales_rate": 12,
      "profit_rate": 15
    },
    "enterprise_product": {
      "rd_rate": 35,
      "cac_rate": 40,
      "cogs_rate": 15,
      "csm_rate": 10
    }
  }
}
```

## 5. 项目历史页 / 详情页详细规格

### 5.1 前置条件

- 项目必须已完成实施成本评估并持久化到 `projects` 表。
- `assessment_details_json` 必须存在，否则不允许生成商务报价。

### 5.2 历史项目列表

列表新增以下能力：

| 项目 | 说明 |
| --- | --- |
| `商务报价状态` 筛选 | 支持按“已生成 / 未生成”过滤 |
| `商务报价状态` 列 | 展示项目是否已生成商务报价 |
| `商务报价总计（万元）` 列 | 展示当前快照计算出的商务报价总额 |
| `商务报价` 按钮 | 打开商务报价弹窗 |
| `导出Excel（商务）` | 导出商务版 Excel；未生成报价时禁用 |

### 5.3 商务报价弹窗

#### 5.3.1 弹窗数据来源

打开弹窗时系统请求：

```http
GET /api/projects/:id/business-quote
```

响应需包含：

- `project_id`
- `project_name`
- `base_cost_wan`
- `default_pricing_mode`
- `default_rates`
- `default_rates_by_mode`
- `business_quote`（若已存在）

#### 5.3.2 弹窗字段

通用字段：

- `报价模式`
- `报价备注`

`B端定制项目` 模式字段：

- 税率
- 管理分摊率
- 销售商务率
- 利润率

`企业级产品` 模式字段：

- 研发成本（R&D）
- 营销与获客成本（CAC）
- 基础设施成本（COGS）
- 客户成功与运维（CSM）

#### 5.3.3 关键展示文案

- 企业级产品模式下，`base_cost_wan` 在弹窗中以 **单客户实施基线** 展示。
- 需通过 Tip 明确说明：这里的“基线”代表原始评估得到的单个客户实施交付直接成本，后续总商业成本池将基于 `COGS + CSM` 占比反推。

#### 5.3.4 实时预览

弹窗底部需实时展示当前模式下的核心结果预览：

`B端定制项目`：

- 管理分摊
- 销售商务
- 税费
- 商务报价总计

`企业级产品`：

- 研发成本（R&D）
- 营销与获客成本（CAC）
- 基础设施成本（COGS）
- 客户成功与运维（CSM）
- 总商业成本池

### 5.4 项目详情页

项目详情页在头部区域展示商务报价摘要：

- 若已生成商务报价，则展示与当前模式对应的关键指标摘要。
- 若未生成商务报价，则展示“未生成商务报价”提示，并引导点击 `商务报价` 按钮。

## 6. 计算与保存规则

### 6.1 数据基线

商务报价保存时，不直接使用列表页缓存金额，而是基于项目保存的 `assessment_details_json` 重新调用原成本计算逻辑，回算实施成本，作为商务报价的输入基线。

### 6.2 B端定制项目公式

```text
管理分摊 = 实施成本 × 管理分摊率
销售商务 = 实施成本 × 销售商务率
利润 = 实施成本 × 利润率
税前小计 = 实施成本 + 管理分摊 + 销售商务 + 利润
税费 = 税前小计 × 税率
商务报价总计 = 税前小计 + 税费
```

### 6.3 企业级产品公式

企业级产品模式下：

- 原评估得到的实施成本视为**单客户实施基线**
- `COGS + CSM` 被视为可变成本占比
- 系统通过可变成本占比反推单客户总商业成本池

```text
可变成本占比 = COGS + CSM
总商业成本池 = 单客户实施基线 / 可变成本占比

R&D 金额 = 总商业成本池 × R&D 占比
CAC 金额 = 总商业成本池 × CAC 占比
COGS 金额 = 总商业成本池 × COGS 占比
CSM 金额 = 总商业成本池 × CSM 占比
```

### 6.4 保存规则

保存接口：

```http
POST /api/projects/:id/business-quote
```

保存规则：

- 每个项目仅保存**一份最新商务报价快照**
- 再次保存时直接覆盖旧快照
- 快照写入 `projects.business_quote_json`

## 7. 商务版 Excel 导出

### 7.1 导出入口

```http
GET /api/projects/:id/export/excel?version=business
```

### 7.2 导出前提

- 项目必须已存在 `business_quote_json`
- 否则前端应禁用导出入口，或在点击时阻止下载

### 7.3 导出内容

商务版 Excel 至少包含两个工作表：

1. `商务报价概览`
2. 模式对应的模块明细页

### 7.4 B端定制项目导出

概览页展示：

- 项目名称 / 描述
- 报价模式
- 实施成本
- 管理分摊率 / 金额
- 销售商务率 / 金额
- 利润率 / 金额
- 税率 / 税费
- 税前小计
- 商务报价总计

模块页展示：

- 模块结构
- 工作量
- 成本占比
- 模块商务报价金额

### 7.5 企业级产品导出

概览页展示：

- 项目名称 / 描述
- 报价模式
- 单客户实施基线
- R&D / CAC / COGS / CSM 的占比与金额
- 非可变成本合计
- 可变成本占比（COGS + CSM）
- 总商业成本池

模块页展示：

- 页签名称为 `模块实施基线明细`
- 金额列名称为 `实施基线（万元）`
- 仅按实施基线分摊到模块，不把总商业成本池错误摊回模块

## 9. 验收标准

### 9.1 配置页

1. `/config` 中存在 `商务报价配置` Tab。
2. 两组配置均使用滑块组件，桌面端两列布局。
3. 企业级产品四项结构合计不为 `100%` 时不可保存。

### 9.2 历史页 / 详情页

4. 历史页操作列存在 `商务报价` 按钮。
5. 打开弹窗时会带出当前项目实施成本基线与对应默认值。
6. 模式切换后，表单字段、说明文案和预览卡片同步切换。
7. 保存后历史页列表可看到 `商务报价状态` 与 `商务报价总计` 更新。
8. 详情页头部可看到商务报价摘要。

### 9.3 导出

9. 未生成商务报价时，`导出Excel（商务）` 不可用。
10. 已生成商务报价后，可导出 `version=business` 的 Excel。
11. 企业级产品模式的模块页必须导出“实施基线”，而不是总商业成本池。

## 10. 备注

本 PRD 描述的是当前已落地的第一阶段能力。后续如需支持多轮报价、审批流、毛利预警或报价版本对比，应在此基础上另开迭代文档。
