# PPA 招标快报小程序实施计划

## 1. 实施目标

在最短时间内交付一版可用于业务验证的微信小程序，完成以下闭环：

1. 微信登录
2. 默认当天招标列表
3. 日期范围筛选
4. 招标详情页
5. 采纳与取消采纳

## 2. 实施原则

1. 先验证业务闭环，不追求通用平台化
2. 先跑通微信分享与采纳，不处理采集与复杂运营能力
3. 数据写入侧预留幂等规则，但推送系统单独建设
4. 富文本公告以可读为先，不做样式保真优化

## 3. 开发任务拆解

## 3.1 阶段一：项目初始化

目标：
- 建立可运行的小程序工程和 CloudBase 环境

任务：
1. 创建 `frontend/ppa_miniapp/` 工程目录
2. 初始化微信小程序项目配置
3. 绑定 CloudBase 环境
4. 配置基础页面路由
5. 建立全局请求与错误处理封装

交付结果：
- 小程序可在开发者工具中启动
- 能成功调用云函数

## 3.2 阶段二：用户登录与身份初始化

目标：
- 完成微信登录和用户记录写入

任务：
1. 创建 `miniapp_users` 集合
2. 实现 `loginUser` 云函数
3. 接入微信登录流程
4. 首次登录补录展示名称
5. 登录态缓存到本地

交付结果：
- 用户首次进入可以完成身份初始化
- 重复进入能识别为同一用户

## 3.3 阶段三：招标列表页

目标：
- 完成当天列表与日期范围筛选

任务：
1. 创建 `tenders` 集合
2. 实现 `getTenderList` 云函数
3. 开发列表页 UI
4. 默认查询当天数据
5. 增加日期范围选择器
6. 列表展示采纳状态
7. 接入分页加载

交付结果：
- 可查看当天列表
- 可按日期范围筛选

## 3.4 阶段四：招标详情页

目标：
- 展示完整详情与公告内容

任务：
1. 实现 `getTenderDetail` 云函数
2. 开发详情页 UI
3. 展示原文链接、摘要、地区、来源平台
4. 增加招标公告渲染
5. 增加富文本失败时的纯文本兜底

交付结果：
- 详情页完整可用
- 招标公告可读

## 3.5 阶段五：采纳与取消采纳

目标：
- 完成核心业务动作

任务：
1. 实现 `adoptTender` 云函数
2. 实现 `cancelTenderAdoption` 云函数
3. 在详情页增加采纳按钮
4. 在已采纳状态下禁用其他用户采纳
5. 在详情页展示采纳人与采纳时间
6. 加入重复点击与并发失败提示

交付结果：
- 一条数据只能被一个人采纳
- 采纳人可取消采纳

## 3.6 阶段六：分享与回归测试

目标：
- 验证微信传播路径和基础稳定性

任务：
1. 配置列表页分享能力
2. 分享参数固定为当天日期
3. 验证他人打开后的登录与列表展示流程
4. 完成关键路径回归测试
5. 整理首版上线检查清单

交付结果：
- 分享路径稳定
- 首版具备业务验证条件

## 4. 数据准备要求

本期虽然不做推送程序，但需要有一批测试数据进入 `tenders` 集合供前后端联调。

建议最小测试数据覆盖：

1. 正常未采纳数据
2. 已采纳数据
3. 重复 `source_item_id` 的更新数据
4. 含富文本公告的数据
5. 截止日期已过的数据

## 5. 联调检查清单

1. 用户首次登录是否成功创建用户记录
2. 列表页默认是否取当天数据
3. 日期范围筛选是否准确
4. 详情字段是否完整
5. 招标公告是否可读
6. 同一条记录能否阻止重复采纳
7. 非采纳人能否被正确阻止取消采纳
8. 分享打开后是否仍能回到当天列表

## 6. 风险与处理

## 6.1 富文本兼容性风险

风险：
- 招标公告 HTML 结构复杂，小程序渲染效果可能失真

处理：
- 保留 `announcement_plain_text` 兜底
- 验收以内容可读为准

## 6.2 并发采纳风险

风险：
- 多用户同时点击采纳造成状态冲突

处理：
- 采纳只允许通过云函数执行
- 服务端返回明确冲突提示

## 6.3 本地 PPA 无法直连风险

风险：
- 当前 PPA 后端未部署线上，无法给微信用户直接访问

处理：
- 首版完全切到 CloudBase
- 推送程序后续单独接入

## 7. 首版完成定义

满足以下条件即可视为首版完成：

1. 微信用户可登录
2. 可查看当天招标列表
3. 可按日期范围筛选
4. 可查看详情
5. 可采纳和取消采纳
6. 可看到采纳人
7. 重复数据不会重复展示
8. 分享出去后其他用户可进入并查看
