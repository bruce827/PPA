# PPA 招标快报小程序 PRD

## 1. 产品概述

PPA 招标快报是一个用于业务验证的微信小程序。用户通过微信分享卡片进入小程序，查看当天或指定日期范围内的招标信息，并对感兴趣的项目执行“采纳”。

该产品不负责采集招标信息，数据由后续独立推送程序写入小程序数据库。首版目标是验证两个核心假设：

1. 通过微信分发，用户是否愿意查看推送的招标信息
2. 用户是否会对感兴趣的招标项目执行“采纳”

## 2. 产品目标

### 2.1 业务目标

1. 让招标信息从 PPA 数据源顺畅分发到微信用户
2. 建立最小可用的“查看 + 采纳”闭环
3. 记录具体由哪个微信用户采纳了某条招标信息

### 2.2 产品目标

1. 用户可通过微信登录进入小程序
2. 用户默认看到当天招标列表
3. 用户可按日期范围筛选招标信息
4. 用户可查看详情与招标公告
5. 用户可对未被采纳的项目执行采纳
6. 用户可取消自己发起的采纳

## 3. 用户定义

### 3.1 目标用户

- 微信用户
- 通过转发分享或主动进入小程序查看招标信息的人

### 3.2 用户权限

- 所有微信登录用户都可查看
- 所有微信登录用户都可对未被采纳的项目执行采纳
- 仅采纳人本人可取消自己的采纳

备注：
- 首版不做白名单
- 后续若要增加访问控制，应在登录后增加授权校验，不改动页面主流程

## 4. 适用场景

1. 某用户收到小程序分享卡片，打开后查看当天招标列表
2. 用户按日期范围筛选查看近期招标
3. 用户点进详情页，阅读项目基本信息与招标公告
4. 用户对感兴趣的项目点击“采纳”
5. 其他用户再次查看该项目时，看到“已被某人采纳”

## 5. MVP 范围

## 5.1 In Scope

1. 微信登录
2. 招标列表页
3. 日期范围筛选
4. 招标详情页
5. 采纳操作
6. 取消采纳操作
7. 采纳人展示
8. 按 `source_item_id` 去重，仅保留最新一条
9. 招标公告原始 HTML 存储
10. 招标公告简化富文本展示
11. 分享后打开“当天招标列表页”

## 5.2 Out of Scope

以下内容本期不做，后续也不纳入本产品范围：

1. 招标信息采集
2. 复杂权限体系
3. 评论
4. 收藏
5. 多维筛选
6. 关键词搜索
7. 推送订阅通知
8. 任务分配
9. 跟进备注
10. 多人协作审批流
11. 独立 CRM 或招投标管理系统能力

## 6. 页面结构

## 6.1 登录/启动页

目标：
- 完成微信身份获取
- 建立小程序用户记录

规则：
- 用户首次进入时完成微信登录
- 若缺少展示名称，可提示补录一个简单显示名
- 登录成功后进入招标列表页

## 6.2 招标列表页

默认行为：
- 默认展示当天招标数据

支持功能：
- 日期范围筛选
- 列表状态展示
- 点击进入详情页
- 分享当前天列表页

列表字段：
- 项目名称
- 发布日期
- 截止日期
- 招标单位
- 预算金额

状态展示：
- 未采纳
- 已被某用户采纳

## 6.3 招标详情页

详情页内容：
- 项目名称
- 发布日期
- 截止日期
- 招标单位
- 预算金额
- 地区
- 来源平台
- 原文链接
- 项目摘要
- 招标公告
- 当前采纳状态
- 采纳人
- 采纳时间

详情页操作：
- 采纳
- 取消采纳

## 7. 业务规则

### 7.1 登录规则

1. 用户必须完成微信登录后才能查看数据
2. 首版不支持匿名查看
3. 用户身份以微信用户身份为准

### 7.2 分享规则

1. 用户可从列表页发起分享
2. 被分享者打开后进入“当天招标列表页”
3. 被分享者仍需完成微信登录

### 7.3 去重规则

1. 招标数据的业务唯一标识为 `source_item_id`
2. 若推送程序两次推送同一 `source_item_id`
3. 系统只保留最新版本
4. 前端列表只展示最新版本

### 7.4 采纳规则

1. 一条招标只能被一个用户采纳
2. 若当前未被采纳，任意登录用户都可点击采纳
3. 采纳成功后必须记录采纳用户身份与采纳时间
4. 若当前已被采纳，其他用户只能查看，不能重复采纳
5. 仅采纳人本人可取消采纳
6. 取消采纳后，状态恢复为未采纳

### 7.5 招标公告展示规则

1. 系统必须保留原始 HTML / 富文本内容
2. 小程序前端优先按富文本展示
3. 若原始样式过于复杂，可降级为结构保留、样式简化
4. 招标公告样式保真不作为首版验收标准

## 8. 数据字段要求

### 8.1 列表页必须字段

- `title`
- `published_at`
- `deadline_at`
- `issuer`
- `budget_amount`

### 8.2 详情页建议字段

- `region`
- `source_platform`
- `source_url`
- `summary`
- `announcement_html`
- `adopt_status`
- `adopted_by_name`
- `adopted_at`
- `updated_at`

## 9. 成功标准

### 9.1 产品成功标准

1. 微信用户可顺利登录并进入招标列表
2. 默认当天列表可正常展示
3. 日期范围筛选可正常工作
4. 详情页可正常打开
5. 采纳与取消采纳状态准确
6. 页面能明确显示当前由谁采纳
7. 重复推送的同一数据不会重复显示

### 9.2 业务验证标准

1. 用户能通过微信分享自然进入小程序
2. 用户能理解“采纳”操作含义
3. 产品能够明确记录采纳人

## 10. 验收标准

1. 微信登录成功后，用户进入列表页
2. 默认筛选为当天
3. 日期范围筛选后的结果正确
4. 列表仅展示指定的 5 个字段
5. 详情页展示额外信息
6. 已采纳项目不能被第二个人再次采纳
7. 采纳人可以取消采纳
8. 页面明确显示采纳人信息
9. 招标公告字段可展示，允许样式降级
10. 重复 `source_item_id` 不产生重复展示
