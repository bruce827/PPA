# 报告生成 (Report Generator)

**版本**: v1.0
**快捷键**: `[REPORT_GEN]` 或 `[RG]`
**输入依赖**: `owner-query-result-{draft_id}.yaml`（由 QUERY_BUILD 生成）

---

## 前置检查

1. 读取 `workflow-state.json` 与 `owner-query-result` 产物。
2. 若 `QUERY_BUILD` 未完成，提示先执行 `[QUERY_BUILD]`。
3. 根据 `report_level` 确定报告粒度（轻量/完整）。

---

## 1. 报告分级

### 1.1 轻量级报告（Level 1）

**适用场景**：简单查询、单一指标

**输出内容**：
- 核心数据（1-3 个关键数字）
- 一句话结论
- 简单图表（可选）

**产物**：`owner-report-lite-{draft_id}.md`

### 1.2 标准报告（Level 2）

**适用场景**：分组对比、趋势分析

**输出内容**：
- 核心发现摘要
- 数据明细表格
- 可视化图表
- 业务建议

**产物**：`owner-report-standard-{draft_id}.md`

### 1.3 完整报告（Level 3）

**适用场景**：问题诊断、综合分析

**输出内容**：
- 执行摘要
- 背景说明
- 分析方法
- 详细发现
- 图表证据
- 结论与建议

**产物**：`owner-report-full-{draft_id}.md`

---

## 2. 智能分级规则

根据以下规则自动判断报告级别：

| 判断条件 | 报告级别 |
|----------|----------|
| 行数 = 1 且 列数 ≤ 2 | Level 1（轻量） |
| 行数 2-10 或 有分组维度 | Level 2（标准） |
| 行数 > 10 或 意图 = diagnosis | Level 3（完整） |

---

## 3. 报告模板

### 3.1 Level 1 模板

```markdown
# 数据查询结果

{一句话业务解读}

## 核心数据

| 指标 | 数值 |
|------|------|
| {指标名} | {数值} |

---
*Generated by Owner Guided Workflow*
```

### 3.2 Level 2 模板

```markdown
# {报告标题}

## 核心发现

- {发现 1}
- {发现 2}

## 数据明细

{数据表格}

## 可视化

{图表占位符或描述}

## 建议

{业务建议}

---
*Generated by Owner Guided Workflow*
```

### 3.3 Level 3 模板

```markdown
# {报告标题}

## 执行摘要

{50 字以内的核心结论}

## 背景

- 分析需求：{用户原始问题}
- 数据范围：{时间范围、筛选条件}
- 分析方法：{分析方法说明}

## 详细分析

### 发现 1：{标题}

{分析内容 + 数据支撑}

### 发现 2：{标题}

{分析内容 + 数据支撑}

## 结论与建议

1. {结论 1}
2. {建议 2}

## 附录：数据明细

{完整数据表格}

---
*Generated by Owner Guided Workflow*
```

### 4.1 文件名

## 8. 多轮追问支持（v1.1 新增）

### 8.1 追问模式判定

若 `workflow-state.json` 中存在 `followup_context`，则为追问模式：

```json
{
  "followup_context": {
    "round": 1,
    "parent_intent": {...},
    "current_intent": {...}
  }
}
```

### 8.2 追问产物命名

追问产生的报告使用以下命名：

- `owner-report-followup-{round}-{draft_id}.md`

示例：
- 第 1 轮追问：`owner-report-followup-1-draft-20260323-000000.md`
- 第 2 轮追问：`owner-report-followup-2-draft-20260323-000000.md`

### 8.3 追问报告模板

```markdown
# 追问分析结果（第 {round} 轮）

## 上一轮查询

{继承的上一轮查询摘要}

## 本轮追问

**追问内容**：{用户追问}
**分析维度**：{新增维度/筛选条件/对比类型}

## 分析结果

{数据表格 + 业务解读}

## 核心发现

- {发现 1}
- {发现 2}

---
*Generated by Owner Guided Workflow - Follow-up Round {round}*
```

### 8.4 追问状态更新

追问模式下，状态更新为：

```json
{
  "workflow_mode": "owner-guided",
  "current_step": "REPORT_GEN",
  "completed_steps": ["OWNER_ENTRY", "INTENT_CLARIFY", "DATA_MAP", "QUERY_BUILD", "REPORT_GEN"],
  "followup_context": {
    "round": 1,
    "parent_intent": {...},
    "current_intent": {...},
    "history": [
      {
        "round": 0,
        "user_input": "...",
        "sql": "...",
        "artifacts": {...}
      }
    ]
  },
  "artifacts": {
    "owner_report": "./outputs/owner-report-followup-{round}-{draft_id}.md"
  }
}
```

### 8.5 追问完成提示

追问报告生成后，输出：

> ✅ **追问分析完成（第 {round} 轮）**
>
> 📄 报告路径：`./outputs/owner-report-followup-{round}-{draft_id}.md`
>
> 你可以：
> - 继续追问（如"再按地区分析一下"）
> - 查看完整历史（使用 `[P]` 查看状态）
> - 开始新问题（直接输入新问题即可）
