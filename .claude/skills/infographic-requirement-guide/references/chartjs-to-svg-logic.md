# Chart.js to SVG Conversion Logic

本文档记录了将基于 Chart.js 的动态 Canvas 图表转换为静态 SVG 的逻辑过程。

## 1. 数据提取 (Data Extraction)
从 HTML 的 `<script>` 标签中提取原始数据数组：
*   **X轴 (年份)**: `years` [1980, 1982, ..., 2021]
*   **左Y轴 (产量)**: `productionData` [216, 189, ..., 65] (最大值设为 350)
*   **右Y轴 (储量)**: `reservesData` [180, 200, ..., 3038] (最大值设为 3500)

## 2. 坐标系映射 (Coordinate System Mapping)
在 SVG 画布 (800x600) 上定义绘图区域：
*   **绘图区范围**: 
    *   X轴: 60px -> 740px (有效宽度 680px)
    *   Y轴: 150px -> 500px (有效高度 350px)
*   **X轴步长**: 680px / (2021 - 1980) ≈ 每年的像素宽度。

## 3. 坐标转换公式 (Transformation Formulas)
将数据点转换为 SVG 的 `(x, y)` 坐标：

### X 坐标 (通用)
$$x = 60 + rac{(年份 - 1980)}{(2021 - 1980)} 	imes 680$$

### Y 坐标 (左轴 - 产量)
$$y = 500 - rac{产量数值}{350} 	imes 350$$
*   例如：1980年产量 216 -> $500 - 216 = 284$
*   对应 SVG 指令: `L60,284`

### Y 坐标 (右轴 - 储量)
$$y = 500 - rac{储量数值}{3500} 	imes 350$$
*   例如：1980年储量 180 -> $500 - 18 = 482$
*   对应 SVG 指令: `L60,482`

## 4. 路径构建 (Path Construction)

### 产量区域图 (Area Chart)
构建封闭路径 `<path>` 并应用填充：
1.  `M60,500` (移动到左下角基点)
2.  `L60,284` (连线到第一个数据点)
3.  ...连接所有计算出的点...
4.  `L740,500` (连线到右下角基点)
5.  `Z` (闭合路径)
6.  应用样式: `fill="url(#prodGradient)"` 和 `filter="url(#oil-effect)"`

### 储量折线图 (Line Chart)
构建开放路径 `<path>` 并应用描边：
1.  `M60,482` (移动到第一个数据点)
2.  ...连接所有计算出的点...
3.  应用样式: `fill="none"` 和 `stroke="#FBBF24"`

通过这种数学映射，我们可以在不运行 JavaScript 的情况下，精确地复刻图表的视觉表现。
